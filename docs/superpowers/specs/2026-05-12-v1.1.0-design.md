Upgrade from v1.0.4 to v1.1.0 encompassing 4 bug fixes, 4 features.

## Bug Fixes

### 1. Version Display Mismatch
- **File:** `src/constants/appMeta.ts:1`
- **Issue:** `APP_VERSION = '1.0.3'` while package/manifest are at `1.0.4`
- **Fix:** Update `APP_VERSION` to `'1.0.4'` (further bumped to `1.1.0` at release)
- **Verification:** Settings > About shows correct version

### 2. Markdown Overflow Scrollbar Mismatch
- **Issue:** Notes card overflow scrollbar uses browser default, not app's glass-morphism styled scrollbar
- **Fix:** Apply existing `--scrollbar-thumb`, `--scrollbar-track` CSS custom properties to `.notes-card .md-preview` overflow container
- **File:** `src/styles/app.css`

### 3. Checkbox Non-Interactive
- **Issue:** Task list checkboxes `- [ ]` / `- [x]` render as styled elements but clicking does nothing
- **Fix:** With new markdown library, render task list items with click handlers that toggle the source markdown
- **File:** `src/components/widgets.tsx` (NotesWidget)

### 4. Incomplete Markdown Rendering
- **Issue:** Only `# H1` and `- [ ]` task lists render; all other markdown falls through to `<p>`
- **Fix:** Replace custom `renderMarkdown` with **`marked`** (markdown parser) + **`DOMPurify`** (HTML sanitizer)
- **Libraries added:** `marked`, `dompurify`, `@types/dompurify`
- **Supported:** headings h1-h6, bold, italic, strikethrough, inline code, code blocks, links, images, blockquotes, ordered/unordered lists, task lists, tables, horizontal rules

## Features

### 1. Multi-Note System

**Data Model:**
```typescript
interface Note {
  id: string;          // crypto.randomUUID()
  title: string;       // default: "dd-mm-yyyy_uniqueTag"
  content: string;     // markdown text
  createdAt: string;   // ISO date
  updatedAt: string;   // ISO date
}
```
**Storage Key:** `ott-notes` → `Note[]`

**UI States:**

| State | Description |
|-------|-------------|
| **List View** | Back button (in card), search bar at top, scrollable table of saved notes sorted by date desc (latest first), delete button per row |
| **Note View** | Title centered at top (click to edit → input field), markdown content below, save button |
| **Today Shortcut** | Button (top-right) jumps to today's note (find or create with `dd-mm-yyyy_uniqueTag`) |

**Title Editing Rules:**
- Click title → transforms to `<input>`
- Empty input → revert to previous title (red border flash)
- Duplicate title (another note has same) → red border, save disabled until unique
- Enter/blur saves title

**Auto-Save:**
- If content changed and unsaved, save automatically with default naming `dd-mm-yyyy_uniqueTag`
- `uniqueTag` = first 8 chars of `crypto.randomUUID()`

**Default Note:** On first load with empty notes array, create one default note with existing `DEFAULT_NOTES` content and today's date title.

### 2. Export/Import Overhaul

**New "Data" Sidebar Section (icon: `Download`/`Upload`):**
- Replaces wallpaper export/import from the Wallpaper section
- Single "Export All Data" button → password prompt → loading spinner + disabled button → encrypted JSON download (async encryption may take time)
- Single "Import Data" button → file picker + password prompt → decrypt + merge

**Export Package:**
```json
{
  "app": "OnTrackTab",
  "version": "1.1.0",
  "exportedAt": "ISO date",
  "data": {
    "tweaks": { ... },
    "settings": { ... },
    "likedWallpapers": [ ... ],
    "notes": [ ... ]
  }
}
```

**Encryption:**
- User provides password at export time
- Web Crypto API: PBKDF2 (100,000 iterations) → derive 256-bit AES-GCM key
- Random 12-byte IV prepended to ciphertext
- Stored as base64: `iv + ciphertext`
- File extension: `.ottdata` (encrypted JSON)

**Decryption:**
- User provides password at import time
- Same PBKDF2 + AES-GCM derivation
- Validate structure after decryption
- Merge: deep merge tweaks/settings, deduplicate liked wallpapers (by dataUrl hash), deduplicate notes (by id)

**API Keys in Export:** Since the encrypted export contains API keys, the password protection ensures they are not stored as plaintext anywhere on disk. The password is never stored — user must remember it.

### 3. MP4 Video Background

**UI:** In Settings > Background, when "Video" type selected:
- Existing: Video URL text input
- New: "Browse" button opens file picker (`<input type="file" accept="video/mp4,.mp4">`)
- On file select: validate size ≤ 50MB (50 * 1024 * 1024 bytes), show error toast if exceeded
- Store file as `blob:` URL via `URL.createObjectURL(videoFile)`
- Save blob URL to `settings.background.videoUrl`

**Rendering:** `StageBackground` already supports `<video>` element — no changes needed.

**Constraints:**
- Blob URLs are session-only (tab lifetime). On reload, the video is lost. This is acceptable for a new tab page.
- Not included in export/import (blob URLs are non-transferable)
- Existing typed-in video URLs continue to work (persisted across sessions)

### 4. Detailed Weather View

**Trigger:** "More" button (or icon) on WeatherWidget card

**UI:** Modal/expanded card showing:
- Current condition + custom weather icon
- Temperature (actual + feels like) 
- Humidity (%)
- Wind speed + direction (if available)
- Visibility (if available)
- UV Index (if available)
- Atmospheric pressure (if available)
- Last updated timestamp
- Tomorrow.io attribution

**API:** Already fetches from tomorrow.io v4 realtime endpoint — all values are in the `values` object. Map additional codes as needed.

**Mapping:**
```
windDirection, visibility, uvIndex, pressureSurfaceLevel
→ weatherCodeLabel for human-readable names
```
